01

Introduction

A road network can be modelled as a weighted graph G = (V, E, w) where vertices V represent junctions or cities, edges E represent road segments, and weights w(e) represent distances or travel times. This page renders such a network on Google Maps alongside an interactive graph view, computes shortest paths using Dijkstra's algorithm, and embeds a GeoGebra applet for the mathematical representation.

Since Java applets are deprecated by modern browsers, the original Java-applet workflow is reimplemented here with the GeoGebra HTML5 API embedded directly via JavaScript.

Traffic-Aware Routing

Replacing distance weights with average travel time turns the same graph into a time-weighted network. Dijkstra now returns the fastest route, demonstrating how the same algorithmic framework supports congestion-aware navigation used by real Google Maps directions.

03

Connectivity & Critical Roads

Identifying bridges (edges whose removal disconnects the graph) highlights critical road segments. For the network shown, removing edge D – F isolates F from the rest of the network if no alternative path of comparable cost exists, illustrating the practical value of graph-theory analysis on road infrastructure.
